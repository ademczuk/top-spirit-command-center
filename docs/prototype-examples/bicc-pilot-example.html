Power BI Desktop BICC pilot

A low-cost, file-export-based pilot that rebuilds the leadership pack in Power BI Desktop before any new licences or live connectors are procured.

Pilot scope

The pilot uses classified file exports from the legacy ERP and bank portal. All data is loaded through Power Query from CSV/XLSX files, transformed inside Power BI Desktop, and visualised in a small set of report pages. No direct database connections, no new Power BI Service licences and no live connectors are needed during the pilot.

Connection lineage

Every visual in the pilot traces back to a named export. This lineage is documented before the model is built so the BICC can reproduce, review and sign off each figure.

Net sales by channel

EXP-ERP-SALES-001 → Power Query transform → fact_Sales → bar + table visuals

Remove discounts and credit notes, convert to net sales, allocate channel from customer master.

Gross profit

EXP-ERP-SALES-001 → Power Query transform → fact_Sales + dim_COGS → KPI cards + variance table

Net sales minus cost of goods sold; duty, freight and excise included where applicable.

Depletions

EXP-WMS-INV-001 → Power Query transform → fact_Movements → line chart by week

Filter outbound warehouse movements, aggregate to cases and 9-litre equivalents.

Open invoices

EXP-ERP-AR-001 → Power Query transform → fact_OpenInvoices → aged-debtor matrix

Calculate days outstanding and age buckets from invoice date and due date.

Guardrails

  • All exports classified T2/T3 remain inside the project workspace; no personal or bank data is published.
  • Report figures carry source-linked audit IDs before they reach steering packs.
  • The pilot is rebuilt as a throw-away learning exercise; no production system depends on it.
  • Go/no-go to a service deployment happens only after the Next-ERP decision is clear.
