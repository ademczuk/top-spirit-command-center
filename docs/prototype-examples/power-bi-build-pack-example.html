Power BI Desktop build pack example

A low-cost, file-export-first pilot for the BICC. No live connectors, no new licences — just classified exports transformed into a source-linked report.

Power BI Desktop M365 E3 Export Layer Source-linked

Build steps

1. Load classified exports

Bring sales_invoice_lines_may.csv, customer_master.csv, and price_list.csv into Power BI Desktop as text/CSV sources. Keep the export ID in a metadata column.

2. Model relationships

Link invoices to customers and products. Avoid merging tables prematurely — keep the star schema simple so the BICC can maintain it.

3. Write DAX measures

Net sales = gross sales − discounts − credit notes. Gross profit = net sales − cost of goods sold. Depletions = sum of cases shipped.

4. Build report pages

Executive summary, channel/brand decomposition, open-invoice ageing, and a source-lineage page showing export IDs and refresh timestamp.

5. Add Approval Envelope

Before any figure is used in a steering decision, a named reviewer signs off on the export classification, measure definition, and source lineage.

6. Capture acceptance evidence

Publish to the BICC workspace, run a trade-meeting walkthrough, and record screenshots plus the business-user acceptance log.

What the BICC gets

  • A working .pbix file built entirely from their own exports.
  • A repeatable refresh routine using SharePoint/OneDrive file drops.
  • A documented measure catalogue tied to the business glossary.
  • A governance checklist for promoting reports to wider use.
