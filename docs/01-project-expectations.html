Project Expectations - Top Spirit (Marussia)

Client expectations preview for the Top Spirit / Angel Software Solutions engagement.
Derived from the project expectations brief, reviewed June 2026.
Scope, timeline and commercials remain subject to an introductory call.

Objective

A process review, automation and AI-literacy engagement that delivers value independent of Top Spirit's two pending decisions - the next ERP and the future BI platform - while building capability inside the team and staying within current tooling and budget.

The Angel Software Solutions approach fits this well: current- and target-state modelling built from exports, AI agents operating inside explicit Approval Envelopes, and a Source-linked Audit Trail.

Deliverables

# Deliverable Description
1 Process mapping End-to-end maps of key processes across all departments, built from exports, reports and interviews, with full source lineage.
2 Process optimisation Target-state redesigns that remove manual and duplicated work, prioritised by effort, value and risk, with quantified before-and-after.
3 AI and literacy LLM / agent workflows under explicit human-approval controls, plus an AI-literacy programme that leaves the capability with the team.
4 APIs and integration Targeted integrations and APIs only where automation justifies the spend; otherwise a file-export approach.
5 Next-ERP advice Vendor-neutral recommendation and migration path, including brownfield versus greenfield trade-offs.
6 Business Glossary A living data dictionary capturing undocumented logic and agreed definitions for key figures: net sales, GP and depletions.
7 Opportunity Catalogue A prioritised list of automation and AI use cases, each with a quantified, traceable business case.
8 Governance model Human-in-the-loop Approval Envelopes mapped to financial and customer-facing controls.
9 BICC support Help the BICC stand up a low-cost Power BI Desktop pilot on the Export Layer.

Working Principles

  • Export Layer first. Work on file exports (Excel, CSV, JSON), not live connectors, so the work is ERP-agnostic and survives any migration.
  • Within budget. Microsoft 365 E3 only; no new licences, software or connectors without an evidence-backed business case.
  • Human oversight. Financial, pricing and customer-facing outputs remain human-validated - never AI-decided.
  • Privacy. GDPR and data classification respected; no user sees data they are not authorised to see.
  • Knowledge transfer. Co-delivered alongside the team, not a black box; all artifacts and capability remain with Top Spirit.
  • Iterative delivery. Wave-based delivery: start small, prove value, then scale.

Phasing

Phase Focus Key outcomes
Phase 0 - Foundations Set the guardrails Data-classification rule; export inventory per system; AI-literacy pilot; first knowledge-capture sessions.
Wave 1 - First value Prove the approach One process mapped end-to-end; one manual report consolidated; one report rebuilt in Power BI Desktop; governance model drafted.
Wave 2 and beyond - Scale Expand and decide Higher-value pilots (document order intake, financial reconciliation); Business Glossary at scale; next-ERP recommendation.

Success Criteria

  • Measured savings. Hours saved per automated task, logged and tracked.
  • Adoption. Rebuilt reports and tools used by the teams in practice.
  • Documentation. Key processes and definitions captured and agreed.
  • Actionable decision. A target-state and migration recommendation for the ERP.
  • Capability. A demonstrable rise in the team's ability to use these tools without external support.

Dependencies

Top Spirit will provide:

  • Data and access. Exports and system access arranged through their team.
  • People. Subject-matter experts for interviews, and an AI Steering Committee sponsor.
  • Counterpart. The BICC as the day-to-day counterpart and owner of the BI track.
