Scale — Value realisation and adoption dashboard spec

Shows whether the engagement is saving time and being used in practice — and proves it. AI/automation can propose the numbers; a human owner approves every value claim before it appears on the dashboard. Each figure traces back to a source export and an audit-trail record.

Project-team working preview. Built on file exports, inside Microsoft 365 E3 (Excel / Power BI Desktop). Companion sample: value-adoption-dashboard-sample.md and the CSV logs in outputs/scale/.

1. Purpose

The dashboard turns the engagement's "Measured savings" and "Adoption" success metrics into reviewed evidence for the AI Steering Committee. It answers three questions every month:

  1. Are we saving time? — hours saved per automated task (AC #1).
  2. Is it being used? — adoption of rebuilt reports and tools by real users (AC #2).
  3. Is the picture current? — refreshed and reviewed at least monthly (AC #3).

No figure is "live" until the named value owner has approved it against its source. This keeps a human in control of the savings story, not just the automations.

2. KPI definitions

KPI Definition Source export / system Owner who approves
Hours saved to date Sum of approved hours saved per live automation/report rebuild. Value-realisation log (outputs/scale/value_realisation_log.csv), reviewed against each baseline time sample. BICC + value owner
EUR value realised Hours saved × loaded hourly rate (client-supplied, illustrative until confirmed). Value-realisation log + HR-confirmed rate. Finance
Automations / tools live Count of use cases in live/adopted status. Opportunity catalogue (docs/08-scale/opportunity-catalogue.md). BICC
Rebuilt reports adopted Reports actually used by the target team in the current period. BICC usage/adoption log (outputs/scale/adoption_log.csv). Report owner
Approval Envelope completion Decisions completed in the human approval queue, with edits/rejections tracked. Approval Envelope log (docs/06-wave-1/governance-approval-envelopes.md). Process owner
Training completion People completing AI-literacy modules, by role. Training attendance/feedback register. AI-literacy lead
Glossary coverage Key terms with an agreed owner, source and review date. Business glossary (docs/08-scale/business-glossary-seed.md). BICC

Every KPI row on the dashboard carries the source export ID and the audit-trail reference that backs it; a KPI with no reviewed source is shown as "Unverified — not counted", never as a savings number.

3. Hours-saved log (AC #1)

One row per automated task. Hours saved are derived, then reviewed — never self-reported by the tool.

Schema (outputs/scale/value_realisation_log.csv):

Field Meaning
opportunity_id Links to the opportunity catalogue (e.g. OPP-001).
automation_name Human-readable use case.
status idea / approved / piloting / live / deferred.
baseline_minutes_per_task Measured manual time from a dated time sample.
automated_minutes_per_task Measured post-change time (incl. human review).
tasks_per_period Volume from a source export (e.g. order count).
period Period the volume covers (e.g. 2026-06).
hours_saved_per_month tasks_per_period × (baseline − automated) / 60.
loaded_hourly_rate_eur Client-confirmed rate (illustrative until HR-confirmed).
eur_value_per_year hours_saved_per_month × 12 × rate.
evidence_ref Time sample / task log that backs the baseline and volume.
source_export_id Export the volume came from (e.g. EXP-ERP-SALES-001).
reviewer Named owner who approved the row.
review_status PENDING_HUMAN_APPROVAL / APPROVED.

Formula — hours_saved_per_month = tasks_per_period × (baseline_minutes_per_task − automated_minutes_per_task) / 60.

Controls: the baseline must be a dated, sampled measurement (not a guess); the automated time must include the human review step so savings are not overstated; a row only counts toward "Hours saved to date" once review_status = APPROVED.

4. Adoption metrics (AC #2)

Adoption is collected from users, not assumed from deployment. A tool that is live but unused scores zero.

Schema (outputs/scale/adoption_log.csv):

Field Meaning
tool_or_report_name Artifact being measured.
audience Target user group.
metric unique viewers / sessions / decisions completed / old workbook retired / terms reviewed.
period Month the value covers.
value Observed value.
target Agreed target for the period.
collection_method How it was captured (BICC log, meeting evidence, owner confirmation, approval log).
source_ref Evidence link / audit reference.
owner Person who confirmed the metric.
Artifact / tool Target users Adoption signal Collection method
Power BI Desktop pilot BICC + report users Refreshed and used in monthly review BICC log / meeting evidence
Consolidated report Report owner / users Old manual workbook retired or reduced Owner confirmation
Business glossary BICC / SMEs Terms searched / updated / reviewed Change log
Approval queue (order intake, reconciliation) Process owners Decisions completed; edits/rejections tracked Approval Envelope log
AI-literacy modules All trained roles Completion by role Attendance register

5. Traceability

  • Each value claim → its source export and the time sample that set the baseline.
  • Each adoption metric → the collection method and audit-log reference.
  • The dashboard renders these as columns, so any number can be challenged and traced in one step. This is the Source-linked Audit Trail applied to the savings story itself.

Dashboard layout mockup: assets/mockups/16-value-adoption-dashboard.mmd.

6. Monthly update routine (AC #3)

Run on a fixed day each month (suggested: first working day), by the BICC or value owner:

  1. Pull the value-realisation log and adoption log from the export layer.
  2. Verify source-linked evidence for any new or changed savings; reject rows without a reviewed baseline.
  3. Confirm adoption metrics with the named owners; flag stalled or zero-use tools.
  4. Recompute hours saved, EUR value and "automations live"; mark unverified rows as not counted.
  5. Update the one-page steering-committee summary (value-adoption-dashboard-sample.md is the template).
  6. Record decisions, approvals and exceptions in the audit trail.

Cadence aligns with the value-realisation tracker template: weekly capture during active pilots, monthly review with the AI Steering Committee, quarterly reconciliation against business-case estimates.

7. What stays a client-run activity

  • Collecting real adoption signals from users (BICC + process owners).
  • Confirming the loaded hourly rate with HR/Finance (replaces the illustrative placeholder).
  • Running the monthly review and recording every approval.

Angel Software Solutions supplies the spec, the schemas, the formulas, the sample and the routine; the client owns the recurring data collection and every approval decision.
