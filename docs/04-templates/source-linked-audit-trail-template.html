Source-linked Audit Trail template

Reusable template + filled example for capturing a traceable record of every AI/agent action and human decision.

Project-team working preview. Every Tier 2 and Tier 3 action must write to this trail.

1. Purpose

The Source-linked Audit Trail is the third Angel Software Solutions pillar. It ensures that every figure, decision and automated action can be traced back to:

  • the source export or trigger file;
  • the AI/agent action and version;
  • the evidence shown to the human approver;
  • the human decision and timestamp;
  • the final outcome.

2. Minimum log schema

Field Description Example
event_id Unique identifier EVT-2026-0001
timestamp_utc When the event was recorded 2026-06-22T09:14:22Z
use_case_id Opportunity / use case reference OPP-008
tier Governance tier 2
trigger_file Source export or input file orders_in/order_2026-06-18.pdf
agent_name Name/version of the AI agent order-intake-agent-v1.2
agent_action What the agent did extract_order_draft
proposal Summary of the AI proposal draft order for CUST-1001, 120x SPI-003, 60x SPI-005
evidence_refs Files/rows/rules used customer_master.csv:row-3; price_list.csv:row-4; stock_availability.csv:row-3
confidence_score Overall confidence (0 - 1) 0.94
approver Human who decided cs-team-lead@topspirit.example
decision approve / edit / reject / escalate approve
decision_notes Free-text rationale Prices and stock confirmed; customer within credit limit
outcome_ref Reference to the committed output drafts/order_2026-06-18_approved.json
gdpr_classification Data classification of trigger Internal

5. Storage and access

  • Store as an append-only CSV or SharePoint list in the Audit library.
  • Read access: AI Steering Committee, BICC, process owners, auditors.
  • Write access: agent service account and approval flows only.
  • Retain for engagement lifetime + 2 years.
