Phase 0 - Data classification rule and export inventory

Illustrative working artifact. Use this before any export is loaded into Angel Software Solutions, Power BI Desktop, an LLM workflow, or an agent prototype.

1. Four-tier classification rule

Tier Label Rule Beverage-distributor examples AI/agent handling
T0 Public Already public or safe for external presentation. Public portfolio categories, generic channel names, public company profile. May be used in public mockups and generic prompts.
T1 Internal Operational context that is not sensitive by itself, but should stay inside the project team. Process names, generic report titles, non-personal workflow notes, anonymised sample layouts. Allowed in internal LLM prompts if no customer/person/pricing detail is included.
T2 Confidential Commercial, financial, operational or customer-sensitive business data. Net sales by customer/brand, GP, depletions by account, price lists, supplier terms, credit notes. Allowed only in approved project workspace; agent outputs require human review and source lineage.
T3 Restricted / GDPR Personal data, special-category data, credentials, highly sensitive contractual or customer-specific material. Named buyer contacts, email addresses, bank details, individual complaints, HR data, credentials/API keys. Do not enter into public tools. Use minimised/redacted extracts only where the user is authorised and processing purpose is recorded.

2. Minimum export inventory fields

Every export used by the engagement gets one row before analysis begins.

Field Description
Export ID Stable ID, e.g. EXP-ERP-ORDER-001.
Source system ERP, CRM, Excel workbook, email mailbox, bank portal, warehouse system.
Export name / filename pattern Human name and file pattern, e.g. sales_orders_YYYYMMDD.csv.
Format Excel, CSV, JSON, PDF, email attachment.
Refresh cadence Daily, weekly, monthly, ad hoc.
Owner Business owner accountable for the export.
Data steward / SME Person who can explain field meaning and undocumented logic.
Classification T0/T1/T2/T3.
GDPR flag None, personal data, financial personal data, mixed/unknown.
Key fields IDs, dates, customer/product keys, values.
Known logic Filters, formulas, manual adjustments.
Downstream use Process map, report rebuild, glossary, opportunity business case.
Source-linked audit ID EXP-* ID used in later artifacts.
Review status Draft / SME reviewed / BICC signed off.

4. Sign-off path

  1. BICC confirms each export owner and availability.
  2. Data owner confirms classification and GDPR flag.
  3. Engagement lead confirms permitted use in AI/agent workflows.
  4. BICC signs off the inventory before Wave 1 automation uses the export.

5. Client completion request

Ask the BICC lead and data owners to complete these fields before any export is used in an AI/agent workflow:

Input needed Why it matters Example / current placeholder
Actual source-system name Prevents generic lineage from becoming fake evidence. Legacy ERP / CRM / bank portal are placeholders.
Export availability and sample date Confirms the file exists and can be refreshed. Daily/monthly cadence is currently TBD.
Export owner + data steward Names who can approve use and explain hidden logic. Finance/BICC, Customer Service, Warehouse, Sales/CRM owner.
Classification and GDPR flag Controls whether the export can enter LLM/agent workflows. Customer refs, contact details and bank data need review.
Retention period and storage location Keeps project copies governed. SharePoint/Teams folder and retention policy TBD.
Permitted use Confirms whether use is allowed for process maps, report rebuilds, prototypes or only manual review. Order-intake and reconciliation require human approval gates.
Sign-off owner Identifies who can close TASK-39. BICC sign-off owner and steering sponsor TBD.

6. Open client inputs

  • Confirm actual source-system names and export availability for each seed inventory row.
  • Fill any TBD cadence, owner/steward, classification, GDPR and retention fields.
  • Confirm whether any export contains named individuals, bank data, customer contacts or pricing terms.
  • Confirm permitted AI/agent usage by export, including whether redacted/minimised samples are required.
  • Confirm BICC sign-off owner and AI Steering Committee sponsor.
