Wave 1 - Order-to-cash process map pack

Use this to run the first end-to-end process mapping exercise. Replace sample evidence with actual exports/interviews after client sessions.

Scope

Order arrival through payment reconciliation for B2B beverage distribution across sales/customer service, finance and logistics.

As-is process

flowchart TD
  A[Customer order arrives by email/PDF/phone] --> B[Customer service manually reads order]
  B --> C[Manual ERP order entry]
  C --> D[Credit and pricing check]
  D --> E[Stock availability check]
  E --> F[Pick, pack and dispatch]
  F --> G[Invoice generated]
  G --> H[Payment received]
  H --> I[Finance manually reconciles payment to invoice]
  I --> J[Manual sales/depletions report updated]

  B:::manual
  C:::manual
  D:::manual
  I:::manual
  J:::manual

As-is pain points to validate

Step Likely pain Evidence to collect
Order reading Key-person knowledge; missed fields in PDFs/emails. Sample order emails/PDFs, SME interview.
ERP order entry Re-keying and typo risk. Order entry timings, error/rework log.
Pricing/credit check Manual lookup and escalation. Pricing sheets, credit hold process.
Reconciliation Bank/export matching is manual. Bank export, open invoice report.
Reporting Copy/paste into management reports. Current workbook and build steps.

To-be target state

flowchart TD
  A[Order file/email lands in Export Layer] --> B[AI extraction assistant drafts order fields]
  B --> C{Approval Envelope: order management review}
  C -->|Approve/edit| D[ERP entry by approved operator or later API/write-back]
  C -->|Reject| B
  D --> E[Automated export validation: customer, pricing, stock]
  E --> F{Approval Envelope: pricing/credit exceptions}
  F -->|Approved| G[Pick, pack, invoice]
  G --> H[Bank/invoice matching assistant suggests reconciliation]
  H --> I{Approval Envelope: finance approval}
  I -->|Approve| J[Post/reconcile and update audit trail]
  J --> K[Power BI/report output refreshed from exports]
