CONTEXT — use this for the design below.

THE CLIENT: an Austrian premium wine, sekt and spirits distributor (B2B). Channels:
supermarkets/retail (off-trade), bars/restaurants/hotels (on-trade), specialist wine
shops, and e-commerce. Core metrics: net sales, gross profit (GP), and depletions (stock
leaving the warehouse to trade). Mid-sized; on a legacy ERP, with an ERP migration AND a
future BI-platform decision both still pending.

THE ENGAGEMENT: a process-review + automation + AI-literacy programme that must (1) work
entirely off file exports (Excel, CSV, JSON) — no live connectors — so it is ERP-agnostic
and survives the migration; (2) stay inside Microsoft 365 E3 with no new software; (3) keep
a human in control of every financial, pricing or customer-facing decision (human-in-the-
loop); (4) be GDPR-compliant and respect data classification. Three signature ideas run
through everything: current- and target-state process modelling; AI agents inside explicit
"approval envelopes"; and a source-linked audit trail (every figure traces back to the
export it came from).

AUDIENCE: the client's executive / AI steering committee, plus one data-literate BI lead.

VISUAL IDENTITY (keep ALL artifacts consistent so they read as one product):
- Frame screens inside a consistent app shell called "Angel Software Solutions": a slim top bar with the
  name, a left nav, a clean content area.
- Palette: near-black navy #0F172A for text/headers, slate #64748B for secondary text,
  very light #F8FAFC backgrounds, white cards, one teal accent #0D9488. Use colour sparingly.
- Typography: a clean sans-serif (Inter / system UI). Generous whitespace.
- Reuse the same terms everywhere: "Export Layer", "Approval Envelope", "Source-linked
  Audit Trail", "Business Glossary", "Opportunity Catalogue".
- Executive-ready, uncluttered, screenshot-quality. No real company name and no real brand
  logos — use "the distributor" or a neutral placeholder.
